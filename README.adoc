== What Is Groove?

Groove is a protocol where independent systems discover each other at runtime,
negotiate capabilities through type-safe interfaces, and compose — or don't —
without either side needing configuration or the other to function.

A groove is the shape cut into the surface of a system. It is passive until
engaged. It guides without constraining. Fit is structural: compatibility is a
property of the manifests, checked at handshake, not a config option.

== Quick Example

[source,bash]
----
# Terminal 1: start the reference provider (this repo)
groove-provider --port 6470

# Terminal 2: probe for groove-aware services
groove probe
----

The provider answers `GET /.well-known/groove` with its capability manifest;
`groove probe` discovers it on both `[::1]` and `127.0.0.1`. Stop the provider —
consumers keep working; the capability disappears gracefully.

== Design Principles

. **Bare by default** — every system works alone, unplugged
. **Structural fit** — compatibility is a manifest property, not a config option
. **Zero configuration** — start services, they find each other
. **Graceful degradation** — losing a groove partner is not an error
. **Linear safety** — connection handles are consumed exactly once
. **Provenance** — every groove connection is attestable

== Repository Map

[cols="1,3"]
|===
| `spec/` | The protocol specification: link:spec/SPEC.adoc[SPEC], link:spec/TRANSPORT.adoc[TRANSPORT], link:spec/IPV6T.adoc[IPV6T], link:spec/MODULARITY.adoc[MODULARITY], link:spec/INNERVATION-SIGNALS.adoc[INNERVATION-SIGNALS], link:spec/CONFORMANCE.adoc[CONFORMANCE] (executable checklist), link:spec/LAYERING.adoc[LAYERING] (aspirational cleave annex)

| `groove-core/` | Shared Rust library: manifest data model, registry, compatibility checking
| `cli/` | `groove` CLI (init / validate / probe / registry / check-compat / mesh)
| `provider/` | `groove-provider` — reference provider + the executable conformance suite
| `clients/` | `browser-extension/` (Firefox MV2) and `js/groove-client.js` (shared client logic)
| `harness/` | Browser-side harness for PanLL-style panels
| `reference/ipv6t/` | GRV6 typed-frame reference implementation (Zig)
| `proofs/` | Idris2 proof modules relocated from gossamer — **not yet decoupled or typechecked**; see link:proofs/README.adoc[proofs/README]
| `docs/` | Papers (`*.tex`), design docs, link:docs/decisions/[decision records], link:docs/REPO-LAYOUT.adoc[repo layout standard]
| `examples/` | Example manifests (JSON and A2ML)
|===

== Status

[cols="1,3"]
|===
| Stage | Working Draft
| Version | 0.2.0
| Media Types | `application/groove+json` (required), `application/groove+a2ml` (optional; registration pending)
| Spec | Drafted (`spec/`); conformance checklist executable (`provider/tests/`)
| CLI | Builds and runs; tested against the reference provider
| Reference provider | Implements discovery + connect/heartbeat/disconnect/mesh lifecycle
| GRV6 (Zig) | Has tests and bench; verified in CI only (zig job)
| Browser extension | Firefox MV2, conformant dialect, generated port table
| Proofs | Relocated, **not typechecked** — task PROOFS-1, allow-fail CI
|===

Consumers named in the papers and registry (Burble, Gossamer, VeriSimDB, Vext,
PanLL) live in their own repositories and are not vendored here.
