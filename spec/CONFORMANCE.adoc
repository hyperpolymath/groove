== Conformance Levels

=== Level 1: Groove-Aware (Minimum)

A system at Level 1:

. MUST expose `GET /.well-known/groove` returning a valid Groove Manifest
. MUST function without any groove partners (bare-first)
. MUST use A2ML format for the manifest
. MUST declare at least one offered capability

=== Level 2: Groove-Connected

A system at Level 2 additionally:

. MUST implement `POST /.well-known/groove/connect`
. MUST implement `POST /.well-known/groove/disconnect`
. MUST implement `GET /.well-known/groove/heartbeat`
. MUST handle graceful degradation when a partner disconnects
. MUST emit `groove:connected` and `groove:disconnected` events
. SHOULD implement local port-probe discovery

=== Level 3: Groove-Proven

A system at Level 3 additionally:

. MUST define capability interfaces in Idris2 with dependent types
. MUST use linear types for connection handles
. MUST generate C headers from the Idris2 ABI
. MUST implement the Zig FFI layer for C-compatible bindings
. MUST generate provenance attestation records for all connections
. SHOULD support the Groove Mesh topology endpoint

=== Level 4: Groove-Native

A system at Level 4 additionally:

. MUST support version negotiation with SemVer constraints
. MUST support capability-based security requirements
. MUST support distributed discovery (mDNS/DNS-SD or BEAM clustering)
. MUST support transitive mesh composition with type-safety proofs
. MUST store provenance in VeriSimDB (when available via groove)

== Compliance Testing

=== Automated Checks

The Groove compliance scanner (Hypatia rule `groove-compliance`) validates:

. Manifest endpoint exists and returns valid A2ML
. Required fields are present (`system.id`, `system.version`, `offers`)
. At least one capability is declared
. Heartbeat endpoint responds within 500ms
. Connection lifecycle endpoints are implemented (Level 2+)
. Idris2 ABI files exist in `src/abi/` (Level 3+)
. Zig FFI files exist in `ffi/zig/` (Level 3+)
. No `believe_me` or `assert_total` in groove ABI code (Level 3+)

=== Self-Test

Every groove-aware system SHOULD include a self-test command:

[source,bash]
----
my-system groove-test
----

This command MUST:

. Verify the manifest is valid
. Verify the system functions without any partners
. Simulate a connection and disconnection
. Verify graceful degradation
. Report the conformance level achieved
